## 2026-03-07 — Phase 0, TB1, Task 3: Create node-client.ts

- Created `packages/api/src/node-client.ts` — typed HTTP client wrapping all node internal endpoints
- Uses Bun-native `fetch` with `AbortSignal.timeout(5000)` for all requests
- Returns discriminated union `NodeResult<T>` — `{ ok: true, data: T }` or `{ ok: false, error, status }`
- Returns `NODE_UNAVAILABLE` (503) on network failures / timeouts
- Wraps all 9 endpoints: `getHealth`, `getBalance`, `getState`, `getMarket`, `getShares`, `getBlocks`, `getBlock`, `submitTransaction`, `distribute`, `referralReward`
- Exports `createNodeClient(baseUrl)` factory and `NodeClient` type
- URL-encodes path parameters (address, marketId) for safety

## 2026-03-07 — Phase 0, TB1, Task 4: Create db/index.ts — SQLite init + users table

- Created `packages/api/src/db/index.ts` — SQLite database module using `bun:sqlite`
- `openDatabase(path)` factory: enables WAL mode, foreign keys, 5s busy timeout, then runs migrations
- `getDb()` singleton accessor with lazy init, uses `DB_PATH` env (default `wpm-api.db`)
- `closeDb()` for graceful shutdown
- `users` table: id (TEXT PK), name, email (UNIQUE COLLATE NOCASE), wallet_address (UNIQUE), wallet_private_key_enc (BLOB), role (DEFAULT 'user'), created_at (INTEGER)
- Email uniqueness is case-insensitive via `COLLATE NOCASE`
- Created `packages/api/tests/db.test.ts` — 7 tests covering WAL mode, foreign keys, column schema, case-insensitive email uniqueness, wallet_address uniqueness, insert/query round-trip, idempotent migration
