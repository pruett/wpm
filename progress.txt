## 2026-03-07 — Phase 0, TB1, Task 5: Create crypto/wallet.ts — wallet key generation + encryption

- Created `packages/api/src/crypto/wallet.ts` — wallet key management module
- `generateWalletKeyPair()` delegates to `@wpm/shared/crypto` `generateKeyPair()` (RSA-2048)
- `encryptPrivateKey(key, secret)` — AES-256-GCM encryption using Web Crypto API (`crypto.subtle`)
- `decryptPrivateKey(enc, secret)` — AES-256-GCM decryption, returns original PEM string
- Key derivation: SHA-256 hash of secret via `Bun.CryptoHasher` → 32-byte AES key
- Random 12-byte IV per encryption; format: `[IV (12 bytes)][ciphertext + GCM auth tag]`
- Decision: did NOT modify `@wpm/shared/crypto` — `@wpm/node` deeply depends on it (all tx/block validation). Used Bun-native APIs in the API layer only.

Gotchas:
- Bun's `Buffer.buffer` returns `ArrayBufferLike` (union of `ArrayBuffer | SharedArrayBuffer`), needs cast to `ArrayBuffer` for Web Crypto API compatibility
- `Bun.CryptoHasher.digest()` returns `Buffer`, not `Uint8Array` — extract underlying `ArrayBuffer` with `.buffer` + `.slice()`

## 2026-03-08 — Phase 0, TB2, Task 1: Create sse/relay.ts — SSE relay

- Created `packages/api/src/sse/relay.ts` — SSE relay module connecting API server to node's event stream
- `SSERelay` class connects to node `GET /internal/events`, parses SSE stream (event/data lines, ignores comments/keepalives)
- Maintains `Map<string, SSEClientEntry>` of connected clients keyed by userId
- Enforces 1 connection per user — `addClient()` closes prior connection before registering new one
- Sends 30s keepalive comments (`: keepalive\n\n`) to each connected client
- Broadcasts parsed node events to all connected API clients via `ReadableStream` controllers
- Auto-reconnects to node on disconnect with exponential backoff (1s → 30s max)
- `createRelay(nodeUrl)` / `getRelay()` singleton pattern matching db module convention
- `close()` method for graceful shutdown: aborts node connection, clears reconnect timer, closes all client streams
