## 2026-03-07 ‚Äî Phase 0, TB1, Task 5: Create crypto/wallet.ts ‚Äî wallet key generation + encryption

- Created `packages/api/src/crypto/wallet.ts` ‚Äî wallet key management module
- `generateWalletKeyPair()` delegates to `@wpm/shared/crypto` `generateKeyPair()` (RSA-2048)
- `encryptPrivateKey(key, secret)` ‚Äî AES-256-GCM encryption using Web Crypto API (`crypto.subtle`)
- `decryptPrivateKey(enc, secret)` ‚Äî AES-256-GCM decryption, returns original PEM string
- Key derivation: SHA-256 hash of secret via `Bun.CryptoHasher` ‚Üí 32-byte AES key
- Random 12-byte IV per encryption; format: `[IV (12 bytes)][ciphertext + GCM auth tag]`
- Decision: did NOT modify `@wpm/shared/crypto` ‚Äî `@wpm/node` deeply depends on it (all tx/block validation). Used Bun-native APIs in the API layer only.
- Created `packages/api/tests/crypto-wallet.test.ts` ‚Äî 8 tests covering key generation, round-trip encrypt/decrypt, random IV uniqueness, wrong secret rejection, tampered/truncated ciphertext rejection

Gotchas:
- Bun's `Buffer.buffer` returns `ArrayBufferLike` (union of `ArrayBuffer | SharedArrayBuffer`), needs cast to `ArrayBuffer` for Web Crypto API compatibility
- `Bun.CryptoHasher.digest()` returns `Buffer`, not `Uint8Array` ‚Äî extract underlying `ArrayBuffer` with `.buffer` + `.slice()`

## 2026-03-07 ‚Äî Phase 0, TB1, Task 6: Create middleware/auth.ts ‚Äî JWT auth

- Created `packages/api/src/middleware/auth.ts` ‚Äî JWT signing, verification, and Hono auth middleware
- `signJwt(payload)` ‚Äî signs JWT with HMAC-SHA256 via Hono's `hono/jwt` utilities, keyed by `JWT_SECRET` env var
- `verifyJwt(token)` ‚Äî verifies signature + expiration, returns typed `JwtUserPayload`
- `authMiddleware` ‚Äî Hono middleware extracting `Bearer` token from `Authorization` header, verifies JWT, sets `c.set("user", payload)` on context
- `JwtUserPayload` type: `sub`, `role` ("user" | "admin"), optional `walletAddress` + `email`, `iat`, `exp`
- Default access token TTL: 15 minutes; custom `exp` supported for admin tokens (24h)
- Returns structured `{ error: { code: "UNAUTHORIZED", message } }` JSON on all 401 paths
- No new dependencies ‚Äî leverages Hono's built-in `hono/jwt` (`sign`/`verify`)
- Created `packages/api/tests/auth.test.ts` ‚Äî 11 tests covering round-trip sign/verify, custom expiry, expired token rejection, tampered token rejection, wrong secret rejection, missing JWT_SECRET, middleware integration (valid token, missing header, malformed header, invalid token, expired token)

## 2026-03-08 ‚Äî Phase 0, TB2, Task 1: Create sse/relay.ts ‚Äî SSE relay

- Created `packages/api/src/sse/relay.ts` ‚Äî SSE relay module connecting API server to node's event stream
- `SSERelay` class connects to node `GET /internal/events`, parses SSE stream (event/data lines, ignores comments/keepalives)
- Maintains `Map<string, SSEClientEntry>` of connected clients keyed by userId
- Enforces 1 connection per user ‚Äî `addClient()` closes prior connection before registering new one
- Sends 30s keepalive comments (`: keepalive\n\n`) to each connected client
- Broadcasts parsed node events to all connected API clients via `ReadableStream` controllers
- Auto-reconnects to node on disconnect with exponential backoff (1s ‚Üí 30s max)
- `createRelay(nodeUrl)` / `getRelay()` singleton pattern matching db module convention
- `close()` method for graceful shutdown: aborts node connection, clears reconnect timer, closes all client streams
